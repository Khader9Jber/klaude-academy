## Flutter's Widget Tree, Built by Conversation

Flutter's widget composition model is elegant but verbose. A simple card with an image, text, and a button is 30+ lines of nested widgets. Claude collapses that verbosity into a single prompt.

## Widget Generation

The core skill: describe what you see, Claude writes the widget tree.

              CircleAvatar(
                radius: 24,

              const SizedBox(width: 16),
              Expanded(
                child: Column(
                  crossAxisAlignment: CrossAxisAlignment.start,
                  children: [
                    Text(
                      name,
                      style: const TextStyle(
                        fontWeight: FontWeight.bold,
                        fontSize: 16,
                      ),
                    ),
                    const SizedBox(height: 4),
                    Text(
                      email,
                      style: TextStyle(
                        color: Colors.grey[600],

                      ),

                ),

        ),

## State Management with Claude

### Riverpod Setup

```bash
claude "Set up Riverpod state management for my Flutter e-commerce app.

Create these providers:
1. authProvider — AsyncNotifier, handles login/logout/token refresh
2. cartProvider — Notifier, manages cart items with add/remove/clear
3. productsProvider — FutureProvider family, fetches product by ID
4. searchProvider — AsyncNotifier, handles search with debounce

Use Riverpod 2.x code generation approach with @riverpod annotation.
Include the necessary build_runner config."
```

```dart
// providers/cart_provider.dart
import 'package:riverpod_annotation/riverpod_annotation.dart';

part 'cart_provider.g.dart';

@immutable
class CartItem {
  final String productId;
  final String name;
  final double price;
  final int quantity;
  final String size;

  const CartItem({
    required this.productId,
    required this.name,
    required this.price,
    required this.quantity,
    required this.size,
  });

  CartItem copyWith({int? quantity}) {
    return CartItem(
      productId: productId,
      name: name,
      price: price,
      quantity: quantity ?? this.quantity,
      size: size,
    );
  }
}

@riverpod
class Cart extends _$Cart {
  @override
  List<CartItem> build() => [];

  void addItem(CartItem item) {
    final index = state.indexWhere(
      (i) => i.productId == item.productId && i.size == item.size,
    );

    if (index >= 0) {
      state = [
        ...state.sublist(0, index),
        state[index].copyWith(quantity: state[index].quantity + 1),
        ...state.sublist(index + 1),
      ];
    } else {
      state = [...state, item];
    }
  }

  void removeItem(String productId, String size) {
    state = state.where(
      (i) => !(i.productId == productId && i.size == size),
    ).toList();
  }

  void clear() => state = [];

  double get total =>
      state.fold(0, (sum, item) => sum + item.price * item.quantity);
}
```

## Custom Painters and Animations

This is where Claude saves the most pain. `CustomPainter` requires knowing Canvas API operations that nobody has memorized.

```bash
claude "Create a CustomPainter that draws a circular progress
indicator with:
- Background circle in light gray (2px stroke)
- Progress arc in a blue-to-purple gradient
- Percentage text centered in the middle (bold, 24sp)
- Rounded stroke caps
- Animated from 0 to target value over 1 second (curved)"
```

```dart
class CircularProgressPainter extends CustomPainter {
  final double progress; // 0.0 to 1.0
  final Color startColor;
  final Color endColor;

  CircularProgressPainter({
    required this.progress,
    this.startColor = const Color(0xFF2563EB),
    this.endColor = const Color(0xFF7C3AED),
  });

  @override
  void paint(Canvas canvas, Size size) {
    final center = Offset(size.width / 2, size.height / 2);
    final radius = (size.shortestSide - 16) / 2;

    // Background circle
    final bgPaint = Paint()
      ..color = Colors.grey[200]!
      ..style = PaintingStyle.stroke
      ..strokeWidth = 8;
    canvas.drawCircle(center, radius, bgPaint);

    // Progress arc with gradient
    final rect = Rect.fromCircle(center: center, radius: radius);
    final gradient = SweepGradient(
      startAngle: -pi / 2,
      endAngle: 3 * pi / 2,
      colors: [startColor, endColor],
    );
    final progressPaint = Paint()
      ..shader = gradient.createShader(rect)
      ..style = PaintingStyle.stroke
      ..strokeWidth = 8
      ..strokeCap = StrokeCap.round;

    canvas.drawArc(
      rect,
      -pi / 2,           // Start from top
      2 * pi * progress,  // Sweep angle
      false,
      progressPaint,
    );
  }

  @override
  bool shouldRepaint(CircularProgressPainter oldDelegate) =>
      oldDelegate.progress != progress;
}

// Usage with animation:
class AnimatedCircularProgress extends StatefulWidget {
  final double targetProgress;
  const AnimatedCircularProgress({
    super.key,
    required this.targetProgress,
  });

  @override
  State<AnimatedCircularProgress> createState() =>
      _AnimatedCircularProgressState();
}

class _AnimatedCircularProgressState
    extends State<AnimatedCircularProgress>
    with SingleTickerProviderStateMixin {
  late AnimationController _controller;
  late Animation<double> _animation;

  @override
  void initState() {
    super.initState();
    _controller = AnimationController(
      duration: const Duration(milliseconds: 1000),
      vsync: this,
    );
    _animation = CurvedAnimation(
      parent: _controller,
      curve: Curves.easeOutCubic,
    );
    _controller.forward();
  }

  @override
  void dispose() {
    _controller.dispose();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return AnimatedBuilder(
      animation: _animation,
      builder: (context, child) {
        final progress = _animation.value * widget.targetProgress;
        return CustomPaint(
          size: const Size(120, 120),
          painter: CircularProgressPainter(progress: progress),
          child: Center(
            child: Text(
              '${(progress * 100).toInt()}%',
              style: const TextStyle(
                fontSize: 24,
                fontWeight: FontWeight.bold,
              ),
            ),
          ),
        );
      },
    );
  }
}
```

You described a progress circle. Claude wrote a `CustomPainter` with gradient shading, a `CurvedAnimation`, and proper dispose cleanup. That's 100 lines of code you didn't need to figure out from the Canvas documentation.

## Key Takeaway

Claude eliminates Flutter's verbosity tax by generating complete widget trees from descriptions. The key is being precise about layout structure (Row vs Column, exact spacing, typography specs) so the generated code matches your design on the first pass. For state management, tell Claude which pattern you use (Riverpod, BLoC, Provider) and it generates idiomatic code for that specific approach. Custom painters and animations — the most painful Flutter features to code from scratch — become one-prompt solutions. Describe the visual result, Claude handles the Canvas API math and animation lifecycle.
