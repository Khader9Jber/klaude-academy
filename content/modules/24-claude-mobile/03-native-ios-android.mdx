## Going Native When It Matters

Sometimes cross-platform isn't enough. You need HealthKit. You need ARKit. You need home screen widgets that update in real-time. You need the platform to feel native because your users will notice if it doesn't.

Claude speaks both Swift and Kotlin fluently.

## SwiftUI View Generation

SwiftUI is declarative and beautiful — but the modifier API surface is enormous. Claude knows it all.

### Basic Views

```bash
claude "Create a SwiftUI ProfileView:
- Navigation title 'Profile'
- Circular profile image (120pt, from URL, placeholder with initials)
- Name (title font, bold)
- Bio (body font, gray, max 3 lines)
- Stats row: Posts | Followers | Following (each with number + label)
- Edit Profile button (bordered, full width)
- List of recent posts (use a ForEach with dummy data struct)

Use @State for the profile data. Include a #Preview."
```

```swift
struct ProfileView: View {
    @State private var profile = UserProfile.sample

    var body: some View {
        ScrollView {
            VStack(spacing: 24) {
                // Avatar
                AsyncImage(url: URL(string: profile.avatarURL)) { image in
                    image.resizable().scaledToFill()
                } placeholder: {
                    Circle()
                        .fill(Color.blue.opacity(0.2))
                        .overlay(
                            Text(profile.initials)
                                .font(.title)
                                .foregroundStyle(.blue)
                        )
                }
                .frame(width: 120, height: 120)
                .clipShape(Circle())

                // Name and Bio
                VStack(spacing: 8) {
                    Text(profile.name)
                        .font(.title2)
                        .fontWeight(.bold)
                    Text(profile.bio)
                        .font(.body)
                        .foregroundStyle(.secondary)
                        .lineLimit(3)
                        .multilineTextAlignment(.center)
                }

                // Stats
                HStack(spacing: 32) {
                    StatItem(count: profile.postCount, label: "Posts")
                    StatItem(count: profile.followerCount, label: "Followers")
                    StatItem(count: profile.followingCount, label: "Following")
                }

                // Edit button
                Button("Edit Profile") {
                    // Navigate to edit
                }
                .buttonStyle(.bordered)
                .frame(maxWidth: .infinity)

                // Recent posts
                LazyVStack(spacing: 16) {
                    ForEach(profile.recentPosts) { post in
                        PostCard(post: post)
                    }
                }
            }
            .padding()
        }
        .navigationTitle("Profile")
    }
}

struct StatItem: View {
    let count: Int
    let label: String

    var body: some View {
        VStack(spacing: 4) {
            Text("\(count)")
                .font(.headline)
                .fontWeight(.bold)
            Text(label)
                .font(.caption)
                .foregroundStyle(.secondary)
        }
    }
}

#Preview {
    NavigationStack {
        ProfileView()
    }
}
```

Claude added the `#Preview` macro, used `AsyncImage` with a proper placeholder, applied correct modifier ordering, and extracted the `StatItem` into a reusable component.

## Kotlin Compose UI

Jetpack Compose is Android's SwiftUI equivalent. Claude handles it just as well.

```bash
claude "Create a Kotlin Compose product listing screen:

TopAppBar with search icon and filter icon.
LazyVerticalGrid with 2 columns of product cards.
Each card: product image (aspect ratio 4:3), name, price,
  rating stars, 'Add to Cart' button.
Pull-to-refresh support.
Loading state with shimmer placeholders.

Use Material 3 theming. Include a ViewModel with StateFlow."
```

```kotlin
@OptIn(ExperimentalMaterial3Api::class)
@Composable
fun ProductListScreen(
    viewModel: ProductListViewModel = hiltViewModel(),
    onProductClick: (String) -> Unit = {}
) {
    val uiState by viewModel.uiState.collectAsStateWithLifecycle()
    val pullRefreshState = rememberPullToRefreshState()

    Scaffold(
        topBar = {
            TopAppBar(
                title = { Text("Products") },
                actions = {
                    IconButton(onClick = { /* search */ }) {
                        Icon(Icons.Default.Search, "Search")
                    }
                    IconButton(onClick = { /* filter */ }) {
                        Icon(Icons.Default.FilterList, "Filter")
                    }
                }
            )
        }
    ) { padding ->
        PullToRefreshBox(
            isRefreshing = uiState.isRefreshing,
            onRefresh = { viewModel.refresh() },
            state = pullRefreshState,
            modifier = Modifier.padding(padding)
        ) {
            when {
                uiState.isLoading -> ShimmerGrid()
                uiState.error != null -> ErrorState(
                    message = uiState.error!!,
                    onRetry = { viewModel.refresh() }
                )
                else -> ProductGrid(
                    products = uiState.products,
                    onProductClick = onProductClick,
                    onAddToCart = { viewModel.addToCart(it) }
                )
            }
        }
    }
}

@Composable
fun ProductGrid(
    products: List<Product>,
    onProductClick: (String) -> Unit,
    onAddToCart: (String) -> Unit
) {
    LazyVerticalGrid(
        columns = GridCells.Fixed(2),
        contentPadding = PaddingValues(16.dp),
        horizontalArrangement = Arrangement.spacedBy(12.dp),
        verticalArrangement = Arrangement.spacedBy(12.dp)
    ) {
        items(products, key = { it.id }) { product ->
            ProductCard(
                product = product,
                onClick = { onProductClick(product.id) },
                onAddToCart = { onAddToCart(product.id) }
            )
        }
    }
}

@Composable
fun ProductCard(
    product: Product,
    onClick: () -> Unit,
    onAddToCart: () -> Unit
) {
    Card(
        onClick = onClick,
        modifier = Modifier.fillMaxWidth()
    ) {
        Column {
            AsyncImage(
                model = product.imageUrl,
                contentDescription = product.name,
                modifier = Modifier
                    .fillMaxWidth()
                    .aspectRatio(4f / 3f),
                contentScale = ContentScale.Crop
            )
            Column(modifier = Modifier.padding(12.dp)) {
                Text(
                    text = product.name,
                    style = MaterialTheme.typography.titleSmall,
                    maxLines = 2,
                    overflow = TextOverflow.Ellipsis
                )
                Spacer(modifier = Modifier.height(4.dp))
                Text(
                    text = "$${product.price}",
                    style = MaterialTheme.typography.titleMedium,
                    color = MaterialTheme.colorScheme.primary,
                    fontWeight = FontWeight.Bold
                )
                Spacer(modifier = Modifier.height(8.dp))
                FilledTonalButton(
                    onClick = onAddToCart,
                    modifier = Modifier.fillMaxWidth()
                ) {
                    Icon(Icons.Default.ShoppingCart, null,
                         modifier = Modifier.size(16.dp))
                    Spacer(Modifier.width(4.dp))
                    Text("Add to Cart")
                }
            }
        }
    }
}
```

## Platform API Integration

This is where native really shines — and where Claude saves the most setup time.

### Camera Access

```bash
claude "Implement camera access for my iOS app (SwiftUI):
1. Check camera permission
2. Request if not determined
3. Show camera picker for photo capture
4. Handle the captured image (save to photos, display preview)
5. Graceful fallback if permission denied (show settings link)

Include the Info.plist key I need."
```

Claude generates the complete flow — permission check, request, camera controller, image handling, and the Settings deep link for denied permissions. Plus the `NSCameraUsageDescription` for Info.plist that you'd forget otherwise.

## Bridging Native and Cross-Platform

When you're adding native modules to a cross-platform app:

```bash
claude "I need a native module for React Native that wraps
iOS HealthKit to read step count and heart rate data.

Generate:
1. The Swift native module with @objc methods
2. The bridge header
3. The TypeScript interface on the RN side
4. A React hook (useHealthKit) that wraps the native calls

Include permission setup for HealthKit."
```

Claude handles the entire bridge — Swift `RCT_EXPORT_METHOD` macros, the bridging header, and the TypeScript types that keep both sides in sync.

## Key Takeaway

Native iOS and Android development with Claude eliminates the API surface area problem. SwiftUI has hundreds of modifiers, Kotlin Compose has dozens of composables, and platform APIs each have their own permission and lifecycle requirements. Claude knows all of these and generates complete implementations — not just the happy path, but permission handling, error states, and platform-specific setup (Info.plist, AndroidManifest). The biggest time savings come from platform API integration (camera, notifications, HealthKit) where forgetting one setup step means hours of debugging, and from App Store metadata where Claude generates ASO-optimized copy in seconds.
