## From Spec to 200 Tests in 5 Minutes

API testing is one of the highest-leverage uses of Claude for QA. Give Claude your OpenAPI spec and it produces a comprehensive test suite that would take days to write manually.

## The OpenAPI-to-Tests Pipeline

```bash
# Step 1: Give Claude your spec
claude -p "Generate a comprehensive API test suite from this OpenAPI spec.

@openapi.yaml

Framework: Vitest + supertest (for HTTP requests)
Base URL: http://localhost:3000/api

For EACH endpoint, generate tests for:
1. Success response — valid request, verify response shape and status code
2. Validation errors — missing required fields, wrong types, invalid values
3. Authentication — request without token, expired token, wrong role
4. Not found — request for nonexistent resource
5. Edge cases — empty strings, very long strings, special characters

Group tests by resource (users, projects, tasks).
Use descriptive test names: 'POST /api/users — should return 422 when email is invalid'

Create a test helper at src/test/api-helpers.ts with:
- getAuthToken(role) — returns JWT for test user with role
- createTestUser(overrides) — creates a user and returns it
- cleanup() — deletes all test data"
```

## Example: Full Endpoint Test Suite

For a single `POST /api/users` endpoint, here is what Claude generates:

```typescript
import { describe, it, expect, beforeAll, afterAll } from 'vitest';
import request from 'supertest';
import { getAuthToken, cleanup } from '../test/api-helpers';

const BASE_URL = 'http://localhost:3000';

describe('POST /api/users', () => {
  let adminToken: string;
  let userToken: string;

  beforeAll(async () => {
    adminToken = await getAuthToken('admin');
    userToken = await getAuthToken('user');
  });

  afterAll(async () => {
    await cleanup();
  });

  describe('success cases', () => {
    it('should create a user with valid data (201)', async () => {
      const response = await request(BASE_URL)
        .post('/api/users')
        .set('Authorization', `Bearer ${adminToken}`)
        .send({
          email: 'newuser@test.com',
          name: 'Test User',
          role: 'user',
        });

      expect(response.status).toBe(201);
      expect(response.body).toMatchObject({
        id: expect.any(String),
        email: 'newuser@test.com',
        name: 'Test User',
        role: 'user',
        createdAt: expect.any(String),
      });
    });

    it('should return the created user without password hash', async () => {
      const response = await request(BASE_URL)
        .post('/api/users')
        .set('Authorization', `Bearer ${adminToken}`)
        .send({
          email: 'nopass@test.com',
          name: 'No Password Visible',
          role: 'user',
        });

      expect(response.body).not.toHaveProperty('password');
      expect(response.body).not.toHaveProperty('passwordHash');
    });
  });

  describe('validation errors', () => {
    it('should return 422 when email is missing', async () => {
      const response = await request(BASE_URL)
        .post('/api/users')
        .set('Authorization', `Bearer ${adminToken}`)
        .send({ name: 'No Email', role: 'user' });

      expect(response.status).toBe(422);
      expect(response.body.error).toMatchObject({
        code: 'VALIDATION_ERROR',
        details: expect.arrayContaining([
          expect.objectContaining({ field: 'email' }),
        ]),
      });
    });

    it('should return 422 when email format is invalid', async () => {
      const response = await request(BASE_URL)
        .post('/api/users')
        .set('Authorization', `Bearer ${adminToken}`)
        .send({ email: 'not-an-email', name: 'Bad Email', role: 'user' });

      expect(response.status).toBe(422);
    });

    it('should return 409 when email already exists', async () => {
      // Create first user
      await request(BASE_URL)
        .post('/api/users')
        .set('Authorization', `Bearer ${adminToken}`)
        .send({ email: 'dupe@test.com', name: 'First', role: 'user' });

      // Try to create duplicate
      const response = await request(BASE_URL)
        .post('/api/users')
        .set('Authorization', `Bearer ${adminToken}`)
        .send({ email: 'dupe@test.com', name: 'Second', role: 'user' });

      expect(response.status).toBe(409);
      expect(response.body.error.code).toBe('DUPLICATE_EMAIL');
    });

    it('should return 422 when role is invalid enum value', async () => {
      const response = await request(BASE_URL)
        .post('/api/users')
        .set('Authorization', `Bearer ${adminToken}`)
        .send({ email: 'a@b.com', name: 'Bad Role', role: 'superadmin' });

      expect(response.status).toBe(422);
    });

    it('should return 422 when name exceeds 100 characters', async () => {
      const response = await request(BASE_URL)
        .post('/api/users')
        .set('Authorization', `Bearer ${adminToken}`)
        .send({ email: 'long@test.com', name: 'A'.repeat(101), role: 'user' });

      expect(response.status).toBe(422);
    });
  });

  describe('authentication', () => {
    it('should return 401 when no token provided', async () => {
      const response = await request(BASE_URL)
        .post('/api/users')
        .send({ email: 'noauth@test.com', name: 'No Auth', role: 'user' });

      expect(response.status).toBe(401);
    });

    it('should return 403 when non-admin tries to create user', async () => {
      const response = await request(BASE_URL)
        .post('/api/users')
        .set('Authorization', `Bearer ${userToken}`)
        .send({ email: 'forbid@test.com', name: 'Forbidden', role: 'user' });

      expect(response.status).toBe(403);
    });
  });

  describe('edge cases', () => {
    it('should trim whitespace from email', async () => {
      const response = await request(BASE_URL)
        .post('/api/users')
        .set('Authorization', `Bearer ${adminToken}`)
        .send({ email: '  spaces@test.com  ', name: 'Spaces', role: 'user' });

      expect(response.status).toBe(201);
      expect(response.body.email).toBe('spaces@test.com');
    });

    it('should handle unicode characters in name', async () => {
      const response = await request(BASE_URL)
        .post('/api/users')
        .set('Authorization', `Bearer ${adminToken}`)
        .send({ email: 'unicode@test.com', name: 'Jose Garcia', role: 'user' });

      expect(response.status).toBe(201);
      expect(response.body.name).toBe('Jose Garcia');
    });

    it('should reject request with empty body', async () => {
      const response = await request(BASE_URL)
        .post('/api/users')
        .set('Authorization', `Bearer ${adminToken}`)
        .send({});

      expect(response.status).toBe(422);
    });
  });
});
```

That is 12 tests for a single endpoint. Across 15 endpoints, you get 180+ tests.

## Schema Validation Tests

Test that every response matches the documented contract:

```bash
"Generate contract tests that verify EVERY API response matches 
the OpenAPI spec schemas.

@openapi.yaml

For each endpoint and status code:
1. Make the request
2. Validate the response body against the JSON Schema from the spec
3. Check required fields are present
4. Check field types match
5. Check enum values are valid
6. Check nested objects match their schemas

Use ajv for JSON Schema validation.
Create a helper: validateResponse(response, schemaName) 
that throws with a clear message on schema mismatch."
```

## Response Time Assertions

```bash
"Add response time assertions to the API test suite.

Create a performance test file at src/__tests__/api-performance.test.ts.

For each endpoint:
- GET requests: must respond within 200ms
- POST/PUT requests: must respond within 500ms
- Search/query endpoints: must respond within 1000ms

Use supertest with response time measurement.
Run each test 5 times and assert on the average.
Flag any endpoint that exceeds its threshold as a warning 
(not a hard failure — just log it)."
```

## Testing Pagination

```bash
"Generate pagination tests for all list endpoints.

Endpoints: GET /api/users, GET /api/projects, GET /api/tasks

Test:
1. Default pagination (page 1, limit 20)
2. Custom page size (limit=5)
3. Second page (page=2)
4. Last page (verify correct count)
5. Page beyond last (should return empty array)
6. Page 0 (should return error or default to page 1)
7. Negative page number
8. Very large page size (limit=10000 — should cap at max)
9. Total count accuracy across pages
10. Sort order is consistent across pages"
```

## Generating Tests from Postman Collections

If you have existing Postman collections:

```bash
"Convert this Postman collection export to a Vitest API test suite.

@postman-collection.json

For each request in the collection:
1. Extract the method, URL, headers, and body
2. Generate a test that sends the same request
3. Assert on the expected status code
4. Assert on the response body shape
5. Add the Postman test scripts as assertions

Group by folder structure from the collection.
Add proper setup/teardown for auth tokens."
```

## Key Takeaway

API testing with Claude scales dramatically when you start from an OpenAPI spec. Claude can generate tests for every endpoint, status code, validation rule, and edge case documented in your spec — and then add the cases that are implied but not documented. Use contract testing to verify response shapes, pagination tests to catch boundary bugs, and response time assertions to catch performance regressions. The combination of a well-documented API spec and Claude's test generation produces a comprehensive test suite in minutes that would take days to write by hand.
