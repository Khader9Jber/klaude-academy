## You Made It

Thirteen modules. From "What is Claude?" to designing multi-agent systems and building cloud automation. If you've worked through every lesson, done the exercises, and built things along the way — you know more about Claude Code than the vast majority of people using it.

Let me be honest about something: when I started using Claude Code, I was the person who typed `claude` and then said "help me write a function." That's it. No CLAUDE.md, no custom commands, no settings.json. Just raw prompts in a terminal. It worked, sort of, in the same way that using a professional camera in full auto mode "works." You get results, but you're leaving 80% of the capability on the table.

The difference between then and now isn't that I learned more commands. It's that I stopped thinking of Claude Code as a chatbot in my terminal and started thinking of it as a **development platform** I build on top of. That mental shift is what this entire course has been about.

## The Complete Skill Tree

Let's look back at everything you've learned — and be honest about which parts clicked and which ones you might want to revisit.

### Foundation Arc (Modules 1-4)

You started here — understanding what Claude is, how tokens work, how to install Claude Code, and how to navigate the interface.

**Key skills acquired**:
- Understanding Claude's model family (Opus, Sonnet, Haiku) and when to use each
- Installing and authenticating Claude Code
- Using slash commands (/help, /model, /effort, /compact)
- Keyboard shortcuts and session navigation
- Effort levels and extended thinking

**The skill that matters most from this arc**: Model switching. I still see experienced users leaving the model on Sonnet for everything. Switching to Haiku for a batch rename saves real money over time, and switching to Opus for a tricky debugging session saves real *hours*.

### Practitioner Arc (Modules 5-8)

You leveled up — configuring Claude for real projects, managing context like a professional, integrating Git workflows, and connecting external tools.

**Key skills acquired**:
- Writing production CLAUDE.md files
- The 5-level CLAUDE.md hierarchy
- Three layers of settings.json
- Tool permissions with glob patterns
- Context window management (80% compaction rule)
- Session naming, forking, and rewind
- Environment variables (LSP, thinking tokens, brief mode)
- Auto-memory management
- Git workflows (/commit, --from-pr, /install-github-app)
- Custom slash commands with $ARGUMENTS
- Advanced commands with shell injection
- Headless mode (claude -p)
- Pipe processing with stdin/stdout
- MCP protocol fundamentals
- GitHub, database, and multi-server MCP setups
- Transport types and configuration scopes

**The skill that matters most from this arc**: CLAUDE.md. Nothing else comes close. A well-written CLAUDE.md transforms Claude from a general-purpose assistant into a team member who knows your project. If you only do one thing from this entire course, write a CLAUDE.md for your main project.

## The Graduation Checklist

Here's the real test. Not multiple choice questions — can you actually *do* these things? Go through this list and be honest with yourself about which ones you'd need to look up.

### Essentials (Everyone should nail these)

- [ ] Write a CLAUDE.md that makes Claude productive in your project from the first message
- [ ] Configure settings.json with appropriate tool permissions (not just `allowAll`)
- [ ] Manage context effectively — compact before 80%, one task per session
- [ ] Use /commit for well-structured, conventional commits
- [ ] Switch models and effort levels based on task complexity
- [ ] Use /help when you forget a command (no shame in this — I still do it)

### Practitioner (Daily workflow confidence)

- [ ] Create 3+ custom slash commands for your most common workflows
- [ ] Set up at least one MCP server (GitHub MCP is the easiest starting point)
- [ ] Use headless mode (`claude -p`) in a script or CI pipeline
- [ ] Process data through pipes (`cat data.json | claude -p "transform this"`)
- [ ] Enable and use LSP for language-aware completions
- [ ] Use session management (naming, forking, resuming) effectively

### Power User (Building on top of Claude Code)

- [ ] Build a PostToolUse auto-format hook that runs Prettier/ESLint on file writes
- [ ] Create a PreToolUse safety guard that blocks writes to production config files
- [ ] Set up notification hooks (Slack, Discord, or system notifications)
- [ ] Define a custom sub-agent with specific permissions and role
- [ ] Create a reusable skill with progressive disclosure
- [ ] Use /plan before complex implementations, then /execute
- [ ] Use /fork for parallel debugging (try two approaches simultaneously)

### Expert (Multiplying your team's effectiveness)

- [ ] Integrate Claude Code in CI/CD (PR reviews, code quality checks)
- [ ] Build a custom MCP server that connects Claude to an internal tool
- [ ] Design a multi-agent workflow (lead agent + specialist sub-agents)
- [ ] Set up /schedule for recurring automation (daily dependency checks, weekly reports)
- [ ] Build a custom CLI tool using `claude -p` with JSON output schemas

If you can check everything in Essentials and Practitioner without looking anything up, you're ahead of 90% of Claude Code users. If you can check the Power User and Expert items too, you're building systems that compound — each one makes everything else more effective.

## What to Build Next

Don't try to build everything at once. Here's a realistic timeline that builds on itself.

### Today (30 minutes)

1. **Add CLAUDE.md to your main project.** Start with the basics: tech stack, key directories, common commands, coding conventions. You can expand it over time. Even a 20-line CLAUDE.md is transformative.
2. **Configure settings.json** with appropriate permissions and an auto-format hook.
3. **Create 2-3 custom commands** for workflows you do every day. If you create PR reviews multiple times a week, make a `/review` command. If you write a certain kind of test often, make a `/test` command.

### This Week

4. **Set up GitHub MCP server.** This lets Claude create issues, review PRs, check CI status, and manage your GitHub workflow without leaving the terminal. It takes 5 minutes to configure and saves hours per week.
5. **Create a security-reviewer agent.** A sub-agent with a specific mandate: review code changes for security issues. Give it a focused system prompt and let it run as part of your pre-commit workflow.
6. **Set up notification hooks.** Get a Slack or Discord ping when Claude finishes a long task or needs your approval. You shouldn't have to babysit the terminal.

### This Month

7. **Integrate Claude Code in your CI/CD pipeline.** Start simple: have it review PRs for code quality. Then expand to security checks, documentation updates, and test coverage analysis.
8. **Build a custom MCP server** for an internal tool your team uses. Even a simple one that wraps 3-4 API endpoints makes Claude dramatically more useful for your specific context.
9. **Set up /schedule for daily dependency checks.** Let Claude run `npm audit` every morning and alert you if there are critical vulnerabilities. Set-and-forget automation.
10. **Create a PR review skill.** A reusable skill that codifies your team's code review checklist, style preferences, and common gotchas.

### Ongoing

11. **Iterate on your CLAUDE.md** as the project evolves. Add new conventions, update architecture descriptions, document gotchas your team discovers.
12. **Add agents and skills as patterns emerge.** When you find yourself giving Claude the same instructions repeatedly, that's a sign you need a command, skill, or agent.
13. **Optimize token usage** by reviewing /insights periodically. You'll find sessions where switching models earlier or compacting sooner would have saved significant tokens.
14. **Share your configuration with your team.** Your CLAUDE.md, settings.json, commands, and hooks are portable. Check them into your repo so every team member benefits.

## The Ecosystem Keeps Growing

Claude Code is evolving rapidly. In the last few months alone, Anthropic has added worktree support, agent teams, the /batch command, /schedule for cloud automation, and MCP elicitation. The pace isn't slowing down.

How to stay current:

- **Update regularly**: Run `claude update` and actually read what changed. New features drop frequently.
- **Check the changelog**: After updates, scan for new slash commands, hook events, and MCP capabilities.
- **Experiment safely**: Use worktrees or isolated branches when trying new features. Don't experiment on main.
- **Watch for new MCP servers**: The MCP ecosystem is expanding quickly. New integrations for databases, APIs, and services appear regularly.
