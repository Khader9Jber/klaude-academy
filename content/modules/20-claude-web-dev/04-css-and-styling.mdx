## CSS Debugging: Claude's Hidden Talent

Every developer has stared at a layout that refuses to center. Or a flex container that overflows. Or a grid that collapses on mobile. These are the moments where Claude Code saves the most time.

## The CSS Debugging Prompt

When something looks wrong, give Claude the context it needs:

```
This layout is broken. The sidebar should be 280px wide and fixed on
the left. The main content should fill the remaining width. But the
content is overlapping the sidebar on screens narrower than 1200px.

Here's the current HTML structure:
[paste the relevant JSX]

Here's the current CSS/Tailwind classes:
[paste or describe]

I want: sidebar fixed at 280px, main content fills remaining width,
on mobile (<768px) the sidebar should collapse to a hamburger menu.
```

Claude will identify the issue — usually a missing `overflow-hidden`, an incorrect `flex` value, or a `width` vs `min-width` problem — and fix it with a clear explanation of why it broke.

### Common CSS Issues Claude Solves Instantly

**"Why is this not centered?"**

```
This div should be centered horizontally and vertically in its parent.
It's currently stuck at the top-left. Parent is the full viewport.
Child is 400px wide, auto height.
```

Claude identifies whether you need `flex items-center justify-center`, `grid place-items-center`, or `absolute top-1/2 left-1/2 -translate-x-1/2 -translate-y-1/2` based on your layout context.

**"Why is there unexpected scrolling?"**

```
The page has a horizontal scrollbar that shouldn't be there.
I can't figure out which element is wider than the viewport.
Here's my layout structure: [paste]
```

Claude will find the element with a fixed width or negative margin that exceeds the viewport and fix it with `overflow-x-hidden` or by adjusting the offending element.

**"Why does this grid break on mobile?"**

```
I have a 3-column grid of cards. On mobile, the cards should stack
vertically. Currently, they're shrinking to tiny columns instead
of stacking. Classes: grid grid-cols-3 gap-4
```

The fix: `grid grid-cols-1 md:grid-cols-3 gap-4`. Claude explains that `grid-cols-3` applies at all breakpoints, so you need the mobile-first approach.

## Tailwind Generation From Descriptions

You can describe a visual design in plain English and Claude will produce the Tailwind classes:

```
Style this section as a hero banner:
- Full viewport height
- Dark gradient background (from gray-950 to gray-900)
- Content centered vertically and horizontally
- Heading: large (text-5xl on desktop, text-3xl mobile), white, bold,
  tracking tight
- Subheading: text-lg, gray-400, max-width 600px, centered
- CTA button: blue background, white text, rounded-lg, padding 12px 32px,
  hover brightens slightly
- Subtle background pattern or noise texture (CSS only)
```

Claude generates:

```tsx
<section className="min-h-screen bg-gradient-to-b from-gray-950 to-gray-900
  flex items-center justify-center px-4">
  <div className="text-center max-w-3xl">
    <h1 className="text-3xl md:text-5xl font-bold text-white tracking-tight">
      Build Faster With Claude
    </h1>
    <p className="mt-4 text-lg text-gray-400 max-w-xl mx-auto">
      Your AI-powered development workflow starts here.
    </p>
    <button className="mt-8 bg-blue-600 hover:bg-blue-500 text-white
      font-medium rounded-lg px-8 py-3 transition-colors">
      Get Started
    </button>
  </div>
</section>
```

Every class is intentional. The responsive prefix on the heading. The `max-w-xl mx-auto` centering the paragraph. The `transition-colors` on the button for smooth hover. Claude knows Tailwind idioms.

## Responsive Design: Mobile-First With Claude

Tailwind is mobile-first, and Claude understands this. Here is how to prompt for responsive layouts:

```
Create a responsive pricing page layout:
- Mobile: single column, cards stacked vertically, full-width
- Tablet (md:): two columns, cards side by side
- Desktop (lg:): three columns with the middle card slightly elevated
- Each card has consistent internal spacing regardless of screen size
- The page has max-width 1200px and is centered
```

Claude generates the responsive grid:

```tsx
<div className="max-w-7xl mx-auto px-4 py-16">
  <div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-8
    items-start">
    <PricingCard plan="starter" />
    <PricingCard plan="pro" className="lg:-mt-4 lg:shadow-xl" />
    <PricingCard plan="enterprise" />
  </div>
</div>
```

The `items-start` prevents cards from stretching to equal height (unless you want that). The negative margin and extra shadow on the middle card creates the "elevated" effect on desktop only.

### Responsive Navigation Pattern

One of the most common requests:

```
Create a responsive navigation bar:
- Desktop: horizontal links with logo on the left, nav items centered,
  CTA button on the right
- Mobile: logo on the left, hamburger button on the right, menu slides
  down when opened
- Use React state for the mobile menu toggle
- Smooth transition on menu open/close
- Close menu when a link is clicked
- Close menu when clicking outside
```

Claude produces a complete responsive nav with proper state management, transitions, and event handling. This is a 50-line component that would take 30 minutes to write and test manually.

## Animation and Transition Prompts

For animations, be specific about timing and easing:

```
Add these animations using Tailwind and CSS:
1. Page sections fade in and slide up 20px when scrolling into view
   (use Intersection Observer, duration 600ms, ease-out)
2. Cards scale up slightly (1.02) on hover with a 150ms transition
3. The mobile menu slides down from the top with a 200ms ease-in-out
   transition
4. Loading spinner: rotating circle, 1s linear infinite
```

Claude adds the appropriate Tailwind classes where possible and creates custom CSS or keyframe animations for anything Tailwind cannot handle natively.

For custom animations in `globals.css`:

```css
@keyframes fade-in-up {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.animate-fade-in-up {
  animation: fade-in-up 600ms ease-out forwards;
}
```

## Matching a Screenshot

When you have a design to match, combine the screenshot with structural description:

```
Match this screenshot [describe or paste image]. Here's what I see:
- Header: sticky, white background, subtle bottom border
- Navigation: logo left, 4 text links centered, avatar dropdown right
- Hero: split layout — text content left (60%), illustration right (40%)
- Heading is dark, about 40px, bold
- Body text is gray, about 16px, max-width ~500px
- CTA: rounded pill button, gradient blue to purple
- Below hero: 3-column feature grid with icon, heading, description
- Spacing: generous, probably 80px between sections
- Overall feel: clean, lots of whitespace, soft shadows

Use Tailwind. React components. Mobile-responsive.
```

This level of description, paired with a screenshot, gets you to 90% accuracy on the first try. One or two refinement passes handle the remaining details.

## Dark Mode With Tailwind

Adding dark mode support is one of the easiest prompts:

```
Add dark mode support to all components in src/components/ui/:
- Use Tailwind's dark: prefix
- Light: white backgrounds, gray-900 text, gray-200 borders
- Dark: gray-900 backgrounds, gray-100 text, gray-700 borders
- Ensure sufficient contrast in both modes (WCAG AA)
- Don't change the component structure, only add dark: classes
```

Claude systematically goes through each component and adds the `dark:` variants. It understands that some colors need more adjustment than others — a `bg-gray-50` in light mode does not simply become `dark:bg-gray-950`, it becomes `dark:bg-gray-800` for proper contrast.

## Key Takeaway

CSS and styling is an area where Claude provides immediate, tangible time savings. For debugging, describe the current behavior, expected behavior, and paste the relevant code. For Tailwind generation, describe layouts in concrete terms — spacing values, colors, breakpoints — not abstract adjectives. Use mobile-first responsive design by default, and be specific about animation timing and easing. Claude knows Tailwind idioms well, so the generated classes are typically clean and idiomatic rather than verbose workarounds.
